# CIP-0002: Installation Mechanism for Target Data Analysis Repos

## Status

- [x] Proposed - Initial idea documented
- [ ] Accepted - Approved, ready to start work
- [ ] In Progress - Actively being implemented
- [ ] Implemented - Work complete, awaiting verification
- [ ] Closed - Verified and complete

## Summary

Design and implement `install.sh` — a bash script that installs the Fynesse framework into any existing data analysis repository with a single command. The script follows the explayner install model: clone/update the fynesse repo locally, copy template stubs and utilities into the target project, write AI assistant rules from the canonical source, and update `.gitignore`.

## Motivation

REQ-0005 requires single-command installation. Without a well-designed install script, adoption requires manual copying of files and understanding of the framework structure. The install script lowers the barrier to adoption to near zero and ensures that AI assistant rules are always up to date when the script is re-run.

The explayner project provides a proven model for this type of lightweight installer. Fynesse's install differs from explayner's in one key respect: the template stubs (`access.py`, `assess.py`, `address.py`) are user content and must not be overwritten on reinstall, whereas explayner's system files (narration.py, compile_narration.py) are never edited by the user and are always overwritten.

## Detailed Description

### Install command

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/lawrennd/fynesse/main/install.sh)
```

Or from a local clone:

```bash
bash ~/lawrennd/fynesse/install.sh
```

### What the script does

**Step 1: Ensure fynesse is available locally**

Clone `github.com/lawrennd/fynesse` to `~/lawrennd/fynesse` if not present, or `git pull --ff-only` if already cloned.

**Step 2: Create fynesse/ module directory (if absent)**

Create `<paper-repo>/fynesse/` directory. Copy template stubs only if the target file does not already exist:
- `access.py` → only if `fynesse/access.py` does not exist
- `assess.py` → only if `fynesse/assess.py` does not exist
- `address.py` → only if `fynesse/address.py` does not exist
- `config.py` → only if `fynesse/config.py` does not exist
- `defaults.yml` → only if `fynesse/defaults.yml` does not exist

This preserves user implementations. Re-running install on an existing project does not overwrite any `fynesse/` module files.

**Step 3: Write AI assistant rules**

Read `rules/fynesse-framework.md` from the cloned fynesse repo and write it to:
- `.cursor/rules/fynesse-framework.mdc` — with Cursor-compatible YAML frontmatter, scoped to `fynesse/**/*.py` and `fynesse/**/*.yml`
- `CLAUDE.md` — insert/replace a fenced section between `<!-- fynesse-framework -->` markers
- `AGENTS.md` — same pattern
- `.github/copilot-instructions.md` — same pattern

AI rules are always overwritten on reinstall (they are system files, not user content). This ensures framework updates propagate to AI assistants.

**Step 4: Update .gitignore**

Add a fenced section to `.gitignore`:
```
# BEGIN fynesse
# Fynesse system files — managed by install.sh, do not commit.
fynesse/config.py
fynesse/defaults.yml
# END fynesse
```

Wait — `config.py` and `defaults.yml` are actually useful to commit (defaults.yml contains project-wide settings). The `.gitignore` should only exclude machine-specific files. Revise:

The `.gitignore` block adds:
```
# BEGIN fynesse
# Machine-specific fynesse config — do not commit.
fynesse/machine.yml
# END fynesse
```

`access.py`, `assess.py`, `address.py`, `config.py`, and `defaults.yml` should all be committed as user content.

### Script structure

The script follows the clean-install philosophy from VibeSafe: idempotent, predictable, system files always updated, user content never overwritten.

Error handling: the script uses `set -euo pipefail`. Each step logs its progress with colour-coded output (graceful fallback if not a terminal).

### Requirements

- `bash` and `git` (hard requirements)
- `python3` (for fenced section replacement in CLAUDE.md/AGENTS.md, using a Python one-liner as in explayner)
- `curl` (for one-line remote install; not required for local install)

## Implementation Plan

1. Write `install.sh` following explayner structure:
   - Step 1: clone/pull fynesse repo
   - Step 2: copy template stubs (skip if exists)
   - Step 3: write AI rules (always overwrite)
   - Step 4: update .gitignore

2. Test on a fresh directory and on an existing project with user-modified stubs

3. Verify reinstall does not overwrite user content

## Backward Compatibility

Initial implementation; no backward compatibility concerns. Future versions should maintain the fenced marker format in CLAUDE.md/AGENTS.md so reinstalls can update the AI rules section.

## Testing Strategy

- Run on an empty directory and verify all expected files are created
- Run on a directory with existing `fynesse/access.py` and verify it is not overwritten
- Run twice and verify idempotency (no duplicate gitignore blocks, no duplicate CLAUDE.md sections)
- Run with `curl` one-liner and verify equivalence with local install
